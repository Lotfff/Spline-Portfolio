- Error type: Vite / Rollup "failed to resolve import" (unresolved import).
- Probable cause: The import "@/components/ui/toaster" in client/src/App.tsx can't be resolved during build. This is usually because either:
  1. The file path doesn't exist or wasn't committed (missing/misnamed file or wrong case), or
  2. The "@" path alias (used to mean client/src) is not configured for Vite (or configured incorrectly), so Rollup can't map "@" to the real folder.

You can inspect the importing file here: https://github.com/Lordx679/Spline-Portfolio/blob/main/client/src/App.tsx

**Solution**

1) Verify the file is present and the import path is correct
   - Confirm the target file exists in the repo at one of these locations (case-sensitive):
     - client/src/components/ui/toaster.tsx
     - client/src/components/ui/toaster.ts
     - client/src/components/ui/toaster.jsx
     - client/src/components/ui/toaster.js
   - If the file is missing, add/commit it to the repository. If the filename uses different casing (e.g. Toaster.tsx), update the import in App.tsx to match exactly.
   - If the import should be a package instead of a local file, verify the package is listed in package.json (and installed) before trying to externalize it.

2) If the file exists but the import still fails, configure the Vite alias so "@" resolves to client/src
   - Edit (or create) client/vite.config.ts (or client/vite.config.js) and add a resolve.alias mapping. Example TypeScript config to add or merge into your existing config:

```ts
// client/vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src'),
    },
  },
});
```

   - If you use JavaScript, the same change applies in vite.config.js (use require and __dirname accordingly).
   - If you use TypeScript path mapping (tsconfig.json), also ensure tsconfig has:
```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```
   - After making these changes, commit them and trigger a Netlify build again.

3) Why not add the module to rollupOptions.external?
   - Vite suggested adding the import to build.rollupOptions.external, but that treats the import as an external dependency and won't bundle it â€” not what you want for a local component. Only use external if the import is an external runtime dependency and you intentionally do not want it bundled.
   - If you ever consider that route, first verify the package is in package.json (or the file was committed), per the instructions above.

Follow these steps (confirm file exists, fix import casing, add/update Vite alias, commit) and your build should resolve the "@/..." import and succeed.
